# Glossary

Agent
:   A configured AI voice persona belonging to a [client](concepts.md#client). See
    [Agents](agents/index.md).

Call
:   One voice conversation — `phone`, `web`, or `test`. See
    [Core concepts](concepts.md#call).

Client
:   An end-customer of yours whose calls an agent answers. See
    [Core concepts](concepts.md#client).

Conditional instruction (trigger)
:   In [Questions & conditions](agents/conversation-design.md#questions-conditions-scenario),
    an instruction that activates only when its condition is met. Can **say**, **ask**,
    and/or **forward**.

Conversation design
:   The mode that controls how an agent runs a call: Questions & forwarding,
    Questions & conditions, or Decision tree. See
    [Conversation design](agents/conversation-design.md).

Decision tree
:   A fully scripted, branching conversation flow (super-admin only). See
    [Conversation design](agents/conversation-design.md#decision-tree-advanced).

Embed
:   A web-call button configuration placed on a website. See
    [Core concepts](concepts.md#embed).

Forwarding rule
:   In simple mode, an "if condition, transfer to number" rule. See
    [Call forwarding](agents/call-forwarding.md).

Intake question
:   A structured piece of information the agent collects from callers. See
    [Configuring an agent](agents/configuration.md#intake-questions).

Knowledge base
:   Free-text an agent uses to answer factual questions about the business. See
    [Configuring an agent](agents/configuration.md#knowledge-base).

Operator
:   Your business account / workspace. See [Core concepts](concepts.md#operator).

Persona
:   A ready-made package of voice, personality, and language for an agent. See
    [Configuring an agent](agents/configuration.md#persona).

Phone number
:   A telephone number routed to an agent for inbound calls. See
    [Core concepts](concepts.md#phone-number).
