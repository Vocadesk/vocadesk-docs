# Getting started

This is the shortest path from a new account to an AI agent answering real calls.
Each step links to the full detail.

## 1. Sign in

Log in to the dashboard with your email and password, or with your connected
identity provider. Your login belongs to an **[operator](concepts.md#operator)** — your
business workspace — where all your clients, agents, and calls live.

## 2. Add a client

A **[client](concepts.md#client)** is an end-customer whose calls an agent will
answer. From **Clients → New client**, give it a name, industry, and contact details.
See [Clients](clients.md).

## 3. Build an agent

Open the client's **Agents** tab and create an
**[agent](agents/index.md)**. In a few minutes you can:

1. Pick a [persona and voice](agents/configuration.md#persona).
2. Write a [greeting](agents/configuration.md#greeting-opening-question) and a short
   [knowledge base](agents/configuration.md#knowledge-base).
3. Add the [intake questions](agents/configuration.md#intake-questions) you want from
   every caller.
4. Choose a [conversation design](agents/conversation-design.md) and, if needed,
   [call-forwarding](agents/call-forwarding.md) rules.

## 4. Test it from your browser

Use **Test call** on the agent page to talk to the agent without touching the phone
network. Test calls aren't billed to the client and are marked as `test`. Iterate on
the persona, questions, and forwarding until it sounds right.

## 5. Go live

- **For phone calls:** attach a [phone number](phone-numbers.md) to the agent.
  Inbound calls to that number are now answered by the agent.
- **For web calls:** create an [embed](embeds.md) and drop its snippet onto your
  website so visitors can talk to the agent from a button.

## 6. Review what happened

Every conversation is captured. Open [Calls](calls.md) to read transcripts, play
recordings, and see the post-call summary, sentiment, and extracted answers.

!!! tip "Do it on a test client first"
    Spin up a throwaway client and agent to learn the flow before configuring a real
    one — nothing here is destructive, and test calls are free.
